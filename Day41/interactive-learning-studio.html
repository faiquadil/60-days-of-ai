Module 1: Classes, Objects, and Instantiation

An OOP architecture relies on a fundamental separation of concern: the Class (the blueprints) and the Object (the actual instances made from the blueprints).

The Analogy: Blueprint vs. Real-World House

Think of a class as an architect's blueprint. The blueprint itself has dimensions, defines windows, and paths. However, you can't live in a blueprint. You build objects (houses) from that blueprint. Each built house has the same layout structure, but its color, occupancy, and status can differ independently.

Building Python Classes: self and Constructors

In Python, the __init__ method acts as the constructor. Whenever you call a class like House(), Python creates an empty container and implicitly passes that new object container into the self parameter of __init__.

class House:
    def __init__(self, color, occupancy):
        # Attribute Initialization
        self.color = color          # Instance Attribute
        self.occupancy = occupancy  # Instance Attribute

# Instantiation
my_house = House("Blue", 4)
print(my_house.color)  # Outputs: Blue

Scope Unique to each individual object. Shared globally among all class instances.
Definition Inside methods using self.name. Directly in the class body, outside methods.

Module 2: Inheritance & Method Overriding

Inheritance allows a new class (child/subclass) to acquire the properties and methods of an existing class (parent/superclass), promoting rapid reuse and scaling.

Concept Deep Dive

When you call a method, Python looks first in your subclass. If it cannot find it, it searches parents recursively. This is dictated by the Method Resolution Order (MRO).

class Vehicle:
    def __init__(self, brand):
        self.brand = brand
    
    def move(self):
        return "Moving dynamically!"

class Car(Vehicle):
    def __init__(self, brand, fuel_type):
        super().__init__(brand) # Invokes parent initializer
        self.fuel_type = fuel_type
        
    def move(self): # Overriding
        return f"{self.brand} is driving fast on {self.fuel_type}!"

Module 3: Encapsulation & Property Decorators

Encapsulation hides the internal data of an object from direct outside access, providing robust data verification boundaries.

Access Modifiers in Python

  • Public: Default attributes accessible anywhere.
  • Protected (_prefix): An API warning to programmers that this property is intended for internal use only.
  • Private (__prefix): Python actively triggers name mangling to prevent clean external mutations.
class BankAccount:
    def __init__(self, owner, balance):
        self.owner = owner
        self.__balance = balance # Private Attribute

    @property
    def balance(self):
        return self.__balance

    @balance.setter
    def balance(self, val):
        if val < 0:
            raise ValueError("Funds cannot be negative!")
        self.__balance = val

Module 4: Magic Methods (Dunder) & Custom Sequences

Dunder Methods (Double Underscore) allow custom objects to integrate seamlessly with native Python operations like slicing, length checks, additions, and string representation.

Common Dunders

  • __str__(self): Readable end-user representations.
  • __repr__(self): Unambiguous developer representations (re-executable Python expression format).
  • __len__(self): Enables integration with len().
  • __add__(self, other): Custom overloading behavior for the + operator.
Interactive Playground (Dunder Overloading)
class Vector: def __init__(self, x, y): self.x = x self.y = y def __repr__(self): return f"Vector({self.x}, {self.y})" def __add__(self, other): return Vector(self.x + other.x, self.y + other.y) v1 = Vector(2, 5) v2 = Vector(3, 1) print(f"Result Vector: {v1 + v2}")
