This comparison demonstrates one of the most transferable principles in prompting: a prompt is a specification, not a request.
Prompt A leaves every assumption open — so the model fills them with statistical defaults (beginner programmer, learning web dev, no time constraints). The output looks like a roadmap, but it's actually a template with your name not filled in.
Prompt B closes those open variables with real signal. Once the model knows your current skills, it skips what you already know. Once it knows your time budget, it can actually pace the days. Once it knows your goal is freelance work, it selects resources and projects that point there.
The three context fields that do the most work, in order:

Current skills — tells the model where to start, not where most beginners start
Goal — defines what "done" looks like, which shapes every milestone backward from it
Available time — the most ignored field, and the one that makes daily tasks actually executable

You can apply this pattern to almost any generation task: the more constraints you provide upfront, the less the model has to hedge across everyone's possible situation — and the more the output fits yours specifically.
