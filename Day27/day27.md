# Day 27 - Prior Authorization Story Simulator

## Objective

Build and explore an interactive HTML-based Prior Authorization Story Simulator to understand how the healthcare prior authorization process works from both the patient's and healthcare provider's perspectives.

## Tools Used

* Claude AI
* HTML
* CSS
* JavaScript
* Visual Studio Code
* GitHub

## What I Learned

* Prior Authorization (PA) is a process where insurance companies review certain medications or treatments before approving coverage.
* Insurance companies verify eligibility, clinical documentation, diagnosis codes, and step therapy history.
* Missing documentation can result in a denial, but it does not necessarily mean the treatment is permanently rejected.
* A Letter of Medical Necessity and supporting clinical documents can be submitted through an appeal.
* Complete and accurate documentation helps reduce delays in patient care.
* Understanding the PA workflow makes the healthcare approval process easier to follow.

## Outcome

Successfully generated the Prior Authorization Story Simulator in HTML, completed all story chapters, explored the approval workflow, and understood both the patient and healthcare operations perspectives.
