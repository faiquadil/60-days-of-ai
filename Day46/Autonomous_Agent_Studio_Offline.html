Agent Roles & Prompt Definitions
1. Planner Agent: Deconstructs software requirements into architecture guidelines, structural boundaries, and strict test verification constraints.
2. Executor Agent: Generates initial functional Python source code and corresponding `unittest` test suites.
3. Evaluator & Critic Agent: Evaluates code against requirements, executes conceptual test pass assessment, identifies edge case bugs, and returns a JSON payload containing `score` (0-100), `zero_critical_bugs` (boolean), `test_pass_rate` (percentage), and detailed `critique`.
4. Improver Agent: Takes code draft, evaluation score, and critic feedback to rewrite and patch the code and test coverage.
5. Final Reviewer Agent: Provides a final holistic code review summary and sign-off once the stop condition triggers.
Architecture Overview & Autonomous Control Loop Mechanics

The Autonomous Agent Studio implements a true dynamic loop governed by runtime stop condition checks rather than static, unrolled agent calls.

Information Flow & State Threading

  • Initialization: The Planner constructs the architectural blueprint from the input specification. The Executor generates Draft 1.
  • State Loop: Evaluator inspects the current code draft and issues a JSON payload containing numerical evaluation metrics and critique points.
  • Stop Check Evaluation: Before entering the Improver step, the runtime engine checks stop rules in order: (1) Zero Critical Bugs & 100% Test Pass Rate, (2) Score Plateau (<3 pts over 2 rounds), and (3) Hard Cap Guardrail.
  • Threaded State: If no stop condition is met, the Improver receives the exact code, previous score, and critique to produce Draft $N+1$. Memory updates track delta changes.

Agent System Prompts & Governance

  • Evaluator Guardrail: Output is structured as JSON to extract literal metrics programmatically without relying on regex guessing.
  • Deterministic Stop Governance: Stopping logic is computed programmatically by the JavaScript orchestrator based on actual live responses from Anthropic Claude API calls.
  • Final Exit Branching: Once a stop rule fires, execution branches away from the Improver directly to the Final Reviewer to generate the audit report.
